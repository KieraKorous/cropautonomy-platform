# Brand and Design System

## Purpose

The project does not yet have established brand assets. This document defines the starting design direction for CropAutonomy and GaiaBots.

The design system should support:

- public landing pages
- authenticated platform dashboards
- technical knowledge base pages
- future robotics telemetry interfaces

Shared DaisyUI theming should be the backbone of the visual system. Prefer theme tokens and reusable component variants over one-off custom styling.

## Shared Visual Direction

The ecosystem should feel:

- field-capable
- industrial
- intelligent
- reliable
- precise
- modern
- robotics-aware
- agricultural

Avoid:

- generic SaaS templates
- playful consumer visuals
- neon gaming aesthetics
- decorative gradients as the main identity
- overly soft startup styling
- fake screenshots that imply unavailable functionality

## Design Inspiration

Use restrained cues from:

- agricultural infrastructure
- mission control interfaces
- robotics telemetry
- GIS dashboards
- field equipment
- environmental monitoring
- industrial systems

## Color Direction

Starting palette direction:

- deep agricultural green
- near-black charcoal
- muted slate
- warm field-neutral tones
- off-white surfaces
- functional amber, red, blue, and green states

The palette should not become one-note green. Use contrast, neutral structure, and functional accents.

## Typography Direction

Typography should be:

- legible outdoors and on mobile
- calm and technical
- not overly futuristic
- suitable for dashboards and long documentation

## Component Direction

Public pages:

- strong first-viewport brand signal
- concise coming-soon positioning
- lead capture
- product/system sections
- restrained motion
- real or generated bitmap visuals when available

Portal:

- dense but readable dashboard layouts
- clear navigation
- strong empty states
- accessible forms
- visible status indicators
- stable layouts for field use

Knowledge base:

- readable technical docs
- device family navigation
- search-ready content structure
- diagrams and tables where useful

## Asset Needs

The project needs:

- CropAutonomy logo
- GaiaBots logo
- GAIA-R concept visual
- GAIA-D concept visual
- shared icon direction
- product screenshots once real screens exist
- field imagery or generated visuals
- device diagrams when hardware direction matures

## Design System Deliverables

Future implementation should define:

- color tokens
- typography scale
- spacing scale
- radius scale
- shadow/elevation rules
- button variants
- form components
- status badges
- cards
- tables
- navigation patterns
- dashboard shells
- landing-page sections
- knowledge-base article components

## DaisyUI Theme Direction

CropAutonomy and GaiaBots should share a consistent DaisyUI-based theme foundation. Brand distinction should come through content, imagery, page composition, and small accent differences rather than entirely separate component systems.

The first implementation should define a shared theme that can be reused across both landing pages and later portal work.
